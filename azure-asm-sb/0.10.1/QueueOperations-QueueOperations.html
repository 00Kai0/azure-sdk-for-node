Class: QueueOperations

QueueOperations

The Service Bus Management API includes operations for managing Service Bus queues. NOTE: An instance of this class is automatically created for an instance of the [ServiceBusManagementClient] ServiceBusManagementClient~ServiceBusManagementClient. See [queues] ServiceBusManagementClient~ServiceBusManagementClient#queues. Initializes a new instance of the QueueOperations class.

Methods

create(namespaceName, queue, callback) → {Stream}

Creates a new queue. Once created, this queue's resource manifest is immutable. This operation is idempotent. Repeating the create call, after a queue with same name has been created successfully, will result in a 409 Conflict error message. (see http://msdn.microsoft.com/en-us/library/windowsazure/jj856295.aspx for more information)

Parameters:
Name Type Description
namespaceName string

The namespace name.
queue ServiceBusQueueCreateParameters

The service bus queue.

Properties
Name Type Argument Description
name string

The name of the queue.
lockDuration string <optional>

Determines the amount of time in seconds in which a message should be locked for processing by a receiver. After this period, the message is unlocked and available for consumption by the next receiver. Settable only at queue creation time: Range: 0 - 5 minutes. 0 means that the message is not locked Default: 30 seconds
maxSizeInMegabytes number <optional>

Specifies the maximum queue size in megabytes. Any attempt to enqueue a message that will cause the queue to exceed this value will fail. You can only set this parameter at queue creation time using the following values: Range: 1 - 1024 (valid values are 1024, 2048, 3072, 4096, 5120) Default: 1*1024 (valid values are 1024, 2048, 3072, 4096, 5120)
requiresDuplicateDetection boolean <optional>

Settable only at queue creation time.* Default for durable queue: false
requiresSession boolean <optional>

Settable only at queue creation time. If set to true, the queue will be session-aware and only SessionReceiver will be supported. Session-aware queues are not supported through REST.* Default for durable queue: false
defaultMessageTimeToLive string <optional>

Depending on whether DeadLettering is enabled, a message is automatically moved to the DeadLetterQueue or deleted if it has been stored in the queue for longer than the specified time. This value is overwritten by a TTL specified on the message if and only if the message TTL is smaller than the TTL set on the queue. This value is immutable after the Queue has been created: Range: 1 second - TimeSpan.MaxValue Default: TimeSpan.MaxValue
deadLetteringOnMessageExpiration boolean <optional>

This field controls how the Service Bus handles a message whose TTL has expired. If it is enabled and a message expires, the Service Bus moves the message from the queue into the queue's dead-letter sub-queue. If disabled, message will be permanently deleted from the queue. Settable only at queue creation time.* Default: false
duplicateDetectionHistoryTimeWindow string <optional>

Specifies the time span during which the Service Bus detects message duplication: Range: 1 second - 7 days Default: 10 minutes
maxDeliveryCount number <optional>

The maximum number of times a message SB will try to deliver before being dead lettered or discarded.
enableBatchedOperations boolean <optional>

Enables or disables service side batching behavior when performing operations for the specific queue. When enabled, service bus will collect/batch multiple operations to the backend to be more connection efficient. If user wants lower operation latency then they can disable this feature.
sizeInBytes number <optional>

Reflects the actual bytes that messages in the queue currently occupy toward the queue's quota.* Range: 0 -MaxTopicSizeinMegaBytes
messageCount number <optional>

Displays the number of messages currently in the queue.
isAnonymousAccessible boolean <optional>

Gets whether anonymous access is allowed.
authorizationRules array <optional>

Gets the authorization rules for the description.
status string <optional>

Gets or sets the current status of the queue (enabled or disabled). When a queue is disabled, that queue cannot send or receive messages.
supportOrdering boolean <optional>

Gets or sets whether the queue supports ordering.
countDetails CountDetails <optional>

Current queue statistics.

Enumerates the queues in the service namespace. The result is returned in pages, each containing up to 100 queues. If the namespace contains more than 100 queues, a feed is returned that contains the first page and a next link with the URI to view the next page of data.
